# MeshCore – Belastungstest Nachrichtenvolumen

## Hintergrund

Am 12. März 2026 fand eine Notfunkübung über MeshCore im Raum Bonn/Köln statt. Dabei erzeugten 11 lokale Teilnehmer insgesamt ca. 50 Nachrichten mit einer durchschnittlichen Länge von ca. 55 Zeichen. Ziel dieses Folgetests ist es, ein vergleichbares Nachrichtenvolumen mit nur **4 Personen** zu erzeugen und dabei die Zustellbarkeit systematisch zu dokumentieren.

## Testparameter

| Parameter | Wert |
|---|---|
| **Datum** | *(einsetzen)* |
| **Startzeit** | 15:30 Uhr |
| **Dauer** | 15 Minuten |
| **Teilnehmer** | 4 Personen (A, B, C, D) |
| **Nachrichten pro Person** | 13 / 13 / 13 / 12 |
| **Nachrichten gesamt** | 51 |
| **Zeichenlänge pro Nachricht** | ca. 55 Zeichen |

## Ablauf

1. Alle vier Teilnehmer sind um **15:30 Uhr** sendebereit.
2. Nach dem **Startschuss** beginnen alle gleichzeitig, ihre vorbereiteten Nachrichten abzusetzen.
3. Die Testphase endet nach **15 Minuten** (spätestens 15:45 Uhr).
4. Jeder Teilnehmer dokumentiert pro Nachricht die Zustellversuche (siehe [Protokollformat](#protokollformat)).
5. Nach Ende des Tests werden die Reports gesammelt und gemeinsam ausgewertet.

:::tip[Tipp für Teilnehmer]
Bereitet euch die Nachrichten vorab als Textbausteine vor, damit ihr sie im 15-Minuten-Fenster schnell per Copy & Paste absetzen könnt.
:::

## Protokollformat

Jeder Teilnehmer notiert pro Nachricht eine **Zahlenkolonne**: Jede Zahl steht für einen Sendeversuch und gibt die Anzahl der Repeats an.

:::info[So liest man die Notation]
- `5` → 1. Versuch: 5 Repeats → sofort erfolgreich
- `0 0 3` → 1. Versuch: 0 Repeats, 2. Versuch: 0 Repeats, 3. Versuch: 3 Repeats
- `0 0 0 0` → 4 Versuche, nie zugestellt (0 Repeats)

Die **Länge der Kolonne** = Anzahl der Versuche. Eine Nachricht gilt als **zugestellt**, sobald mindestens ein Wert > 0 auftritt.
:::

### Beispiel

| Nachricht | Versuche (Repeats) | Bedeutung |
|---|---|---|
| A-01/13 | `5` | 1. Versuch: 5 Repeats → sofort erfolgreich |
| A-02/13 | `0 0 3` | 1. Versuch: 0 Repeats, 2. Versuch: 0 Repeats, 3. Versuch: 3 Repeats |
| A-03/13 | `0 0 0 0` | 4 Versuche, nie zugestellt (0 Repeats) |

## Auswertung

Nach dem Test werden folgende Kennzahlen ermittelt:

- **Zustellrate** – Anteil der Nachrichten, die mindestens 1 Repeat erhalten haben
- **Versuche bis Zustellung** – Durchschnittliche Anzahl Sendeversuche pro erfolgreicher Nachricht
- **Repeat-Durchschnitt** – Mittlere Repeat-Anzahl bei erfolgreichen Zustellungen
- **Vergleich mit Übung vom 12.03.** – Wie verhält sich die Zustellbarkeit bei 4 vs. 11 Teilnehmern?

:::note[Ergebnisse]
Die Ergebnisse werden nach Durchführung des Tests hier ergänzt.
:::
